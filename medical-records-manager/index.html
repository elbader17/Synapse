Características Principales

Diseñado desde cero para resolver los desafíos más críticos en el manejo de información de salud electrónica (EHR).

Control de Acceso (RBAC)

Sistema granular de roles (Médico, Enfermera, Admin, Paciente) que restringe el acceso a endpoints y datos específicos según permisos.

Auditoría Inmutable

Registro completo de quién hizo qué, cuándo y desde dónde. Los registros de auditoría no pueden ser alterados ni eliminados.

Autenticación TOTP 2FA

Implementación estándar RFC 6238. Soporte para Google Authenticator, Authy y códigos de respaldo de un solo uso.

Gestión de Pacientes PII

Separación estricta de Información de Identificación Personal (PII) de los datos clínicos utilizando identificadores seudónimos.

Soft Deletes & Retención

Eliminación lógica de registros para cumplir con políticas de retención de datos médicas, garantizando el derecho al olvido del GDPR.

Rate Limiting

Protección contra ataques de fuerza bruta y abusos de API mediante un algoritmo de ventana deslizante en memoria.

Seguridad de Grado Militar y Cumplimiento Normativo

La privacidad del paciente no es opcional. El sistema encripta toda la información sensible en reposo (At Rest) y en tránsito, garantizando la aprobación de auditorías de salud.

  • HIPAA Compliant (EE. UU.) Control de acceso, registros de auditoría inmutables, seguridad de transmisión y políticas de contraseñas estrictas.
  • GDPR / LGPD (Europa/Brasil) Minimización de datos (PII encriptado por separado), derecho al acceso y borrado lógico con retención de logs.
  • Encriptación AES-256-GCM Campos como Nombre, SSN, Dirección e Historias Clínicas se cifran a nivel de aplicación antes de tocar la base de datos.

Clean Architecture en Go

Separación clara de responsabilidades para un código mantenible, testeable y escalable.

1

Transport Layer

Manejo HTTP con Gin, formato de requests/responses, Middleware de seguridad.

2

Application Layer

Casos de uso, lógica de orquestación, validación de reglas de negocio.

3

Domain Layer

Entidades puras (User, Patient, Record), interfaces de repositorios.

4

Infrastructure Layer

Conexión a PostgreSQL (pgx), servicio Crypto AES-256, Logging.

Referencia Rápida de API

Método Endpoint Descripción Auth
POST /api/v1/auth/login Inicio de sesión (Retorna Temp Token si 2FA)

POST /api/v1/medical-records Crea una nueva historia clínica (Encripta datos)
GET /api/v1/medical-records/patient/:id Obtiene registros (Requiere rol Médico/Admin)
